1. Virtual DOM 이 무엇인지 설명해주세요.
   1. 가상 DOM은 리액트에서 사용되는 개념입니다.
   2. 리액트 파이버라는 객체로 이루어진 가상돔을 리액트는 가지고 있어요
   3. 가상 DOM의 변화를 감지하여 변화가 일어나면 가상 DOM을 업데이트합니다.
   4. 가상 DOM이 업데이트되면 이를 실제 DOM에 반영되게 하는 방식을 채용합니다.
2. JSX에 대해 설명해주세요
   1. JSX는 html을 자바스크립트에서 표현할 수 있게 한 것
   2. 리액트의 것은 아니지만 리액트에서 채용하고 있음
3. 엘리먼트 vs 컴포넌트
   1. 컴포넌트는 props를 받아서 엘리먼트를 반환하게 된다.
   2. 엘리먼트는 한번 생성되면 변하지 않음 ( 불변성 )
4. ref vs state
   1. ref와 state는 구조적으로 크게 다르지 않음
   2. 컴포넌트가 렌더링 되어도 이전의 값을 지니고 있을 수 있음
   3. 단 ref는 해당 값이 바뀌어도 리렌더링을 일으키지 않음
   4. 따라서 dom에 직접접근할때 유용함

# 리액트

- 리액트에서 불변성을 유지하는 이유
  리액트에서 불변성을 유지하면 상태변화가 한번에 일어나도 그 순서를 보장할 수 있다. (사이드이펙트가 없음)
- 코드 스플리팅이란
  - 파일이 너무 커지면 유저 입장에서 오래 기다리게 되기 때문에 분리하는 것
  - React.lazy나 Suspense를 통한 분리를 적용할 수 있음
  - 특히 라우팅 기준으로 이를 나누면 좋음
- React에서 컴포넌트가 언마운트될 때 리소스 해제를 어떻게 처리하나요?
  이벤트 리스너는 클린업 함수를 통해서 직접 제거해 줘야함
  자동으로는 이벤트핸들러 등은 없애주고 setState와 같은 호출을 무시하게 됨
- 트랜스파일링 번들링
  - JSX라는 독특한 문법을 사용하는 리액트의 특성 상 babel을 통해서 브라우저가 이해할 수 있게 변환해줘야함
  - babel을 통해서 폴리필 등도 추가댐
- React의 주요 특징은 무엇인가요?
  - 가상돔을 활용하여 DOM을 제어함
  - JSX를 사용해서 개발자가 좀더 편하게 작업이 가능하다.
  - 단방향 데이터 흐름을 가져서 로직이 복잡해져도 추적이 쉬움
  - 큰 커뮤니티를 가지고 있음
  - 렌더 → 커밋 단계를 수행
- React에서 이벤트 처리 방법을 설명해보세요.
  합성이벤트라는 객체로 처리를 함. 이를 통해 크로스 브라우징 이슈도 해결할 수 있음. 특이한건 리스너가 부착하는 DOM이 아닌 상위에 이벤트가 등록이 한번에 됨. 16→17로 오면서 부착되는 위치가 바뀌었음
- 조건부 렌더링을 구현하는 방법은 무엇인가요?
  말 그대로 조건에 따라 다르게 렌더링 되는 것
  조건문을 활용해서 구현할 수 있음 & 연산자 사용도 가능
- React Router를 어떻게 설정하고 사용하나요?
  react-router-dom을 사용합니다. nextjs의 경우에는 자동적으로 설정이 되어있음
- 컴포넌트 간의 데이터 전달 방법은 어떤 것이 있나요?
  - props, 전역상태관리
- React의 키(key)의 역할은 무엇이며 왜 중요한가요?
  형제 컴포넌트를 구별할 수 있게함. 없다면 렌더링도 비효율적이게 되고 안정성이 보장되지 않을 수 있음
- 컴포넌트 스타일링 방법은 어떤 것이 있나요?
  css,css-in-js, tailwind 등등
- React Hooks는 무엇이며 어떻게 사용하나요?
  - 리액트 컴포넌트안에서만 사용해야 하고
  - 위에 작성해서 순서가 항상 보장되게 해야함
  - hook의 이름은 use로 시작하게 하는것이 규칙
- Redux의 액션, 리듀서, 스토어의 역할을 설명해보세요.
  - ACTION : 말그대로 action을 정의
  - 리듀서 : Action을 받아 새로운 상태를 반환하는 순수 함수
  - 스토어 : 리듀서를 통해 스토어의 상태를 업데이트 함
- Redux Thunk와 Redux Saga의 차이점은 무엇인가요?
  - 둘다 Redux내에서 비동기처리가 가능하게 함
  - thunk : thunk함수를 디스패치 할 수 있게 하는 미들웨어
  - saga : 제너레이터 함수 문법을 기반으로 비동기작업을 관리
  - saga는 비동기작업 자체를 분리할 수 있지만 러닝커브가 있고 좀더 무겁다.
- React에서 성능 최적화를 위한 방법은 어떤 것이 있나요?
  - 코드 스플리팅
  - 메모이제이션
- React의 Portals는 무엇이며 어떻게 사용하나요?
  - react 컴포넌트의 DOM 위치를 변경시켜줄 수 있게 해줌
- React와 SEO (Search Engine Optimization)를 어떻게 연동하나요?
  - robots.txt 설정
  - SSR&SSG와 같이 미리 렌더링을 해두는 방식
  - meta tag 설정하기
  - 링크에 a태그 사용하기
- React의 Error Boundary는 무엇이며 어떻게 사용하나요?
  - 에러가 났을때 fallback을 사용해서 사용자에게 다른 화면을 보여줄 수 있음
  - 에러가 났을때 가장 가까운 Error Boundary에 접근할 수 있음
  - 에러바운더리에서 난 에러나 비동기 함수들은 잡을 수 없어서 try-catch를 써야함
- React의 렌더 프롭 (Render Props) 패턴은 무엇이며 어떻게 사용하나요?
  - render라는 속성에 prop으로 컴포넌트를 반환하는 함수를 넣어서 컴포넌트의 재사용성을 늘려줌
  - 상태를 부모로 올리지 않고도 자식에서 상태를 관리하는 것이 가능하다.
- React와 Webpack의 관계는 무엇이며 어떻게 설정하나요?
  웹 번들링 도구
- React의 Strict 모드는 어떤 역할을 하나요?

  - 가상돔은 브라우저의 계산을 메모리에서 수행할 수 있게 해줌
  - 렌더 → 커밋 단계로 이루어져 있음
